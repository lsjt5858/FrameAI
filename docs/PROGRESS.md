## 当前阶段

当前处于 `Phase 1: MVP 闭环`。

第一阶段目标：跑通“创建项目 -> 上传素材 -> 创建分镜 -> 套用提示词 -> 创建生图任务 -> 保存结果 -> 创建生视频任务 -> 保存结果 -> 标记入选”的最小闭环。

- ❌ 真实 Provider 验收：OpenAI / DashScope / Volcengine 适配已接入，但真实调用仍需要 API Key、公网素材地址和网络环境验证。
- ❌ Figma 精准对齐：当前会话未暴露 Figma MCP 的 `get_design_context/get_screenshot` 工具，页面只能依据已下载设计和截图做高相似实现；接入 Figma MCP 后可继续做像素级校准。

## 技术债与注意事项

- ✅ 前端依赖已具备，`npm run build` 可通过
- ✅ 后端虚拟环境已具备，`python3 -m compileall backend/app` 可通过
- ❌ 当前 mock video 结果是 JSON 占位文件，接真实平台后改为真实视频文件
- ❌ 当前未引入 ORM，后续迁移 PostgreSQL 时可考虑 SQLAlchemy 或 SQLModel
- ❌ 当前任务 worker 随 FastAPI 进程启动，后续可迁移到独立 worker / Celery
- ❌ 真实 OpenAI / DashScope / Volcengine 调用尚未验收，需要 API Key 和可用网络
- ❌ DashScope 图生视频需要公网图片 URL，本地 `storage/` 需配置公开访问地址
- ❌ Volcengine 图生视频需要公网图片 URL，本地 `storage/` 需配置公开访问地址

## 更新规则

每完成一个模块时，按下面顺序更新：

1. 把对应 checklist 从 `❌` 改成 `✅`，如果只是基础版先改成 `✅（基础版）`。
2. 如果整个小模块都完成，同步更新阶段总览状态。
3. 在“完成记录”新增一行，写清楚日期、模块和结果。
4. 如果发现新的风险或遗留问题，补到“技术债与注意事项”。
