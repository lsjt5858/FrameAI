# FrameAI 开发进度

更新时间：2026-05-24

这个文档用于跟踪 FrameAI 的开发进度。完成项统一用 ✅ 表示，未完成项统一用 ❌ 表示，并在“完成记录”里补一条简短记录。

## 状态说明

- `✅`：已完成
- `✅（基础版）`：已完成基础版，后续需要增强
- `❌（进行中）`：正在开发但尚未完整验收
- `❌`：未完成

## 当前阶段

当前处于 `Phase 1: MVP 闭环`。

第一阶段目标：跑通“创建项目 -> 上传素材 -> 创建分镜 -> 套用提示词 -> 创建生图任务 -> 保存结果 -> 创建生视频任务 -> 保存结果 -> 标记入选”的最小闭环。

## 技术债与注意事项

- ❌ 前端依赖安装需要联网访问 npm registry，当前环境尚未完成构建校验
- ❌ 后端需要创建虚拟环境并安装 `backend/requirements.txt`
- ❌ 当前 mock video 结果是 JSON 占位文件，接真实平台后改为真实视频文件
- ❌ 当前未引入 ORM，后续迁移 PostgreSQL 时可考虑 SQLAlchemy 或 SQLModel
- ❌ 当前任务 worker 随 FastAPI 进程启动，后续可迁移到独立 worker / Celery

## 更新规则

每完成一个模块时，按下面顺序更新：

1. 把对应 checklist 从 `❌` 改成 `✅`，如果只是基础版先改成 `✅（基础版）`。
2. 如果整个小模块都完成，同步更新阶段总览状态。
3. 在“完成记录”新增一行，写清楚日期、模块和结果。
4. 如果发现新的风险或遗留问题，补到“技术债与注意事项”。
