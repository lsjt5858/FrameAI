# FrameAI A 方案前端改版设计

## 背景

当前 `frontend/src/App.jsx` 采用单文件、多标签页的工作台结构，已经具备完整的业务流程，但存在以下问题：

- 页面整体偏表单堆叠，缺少工作台感
- 信息层级不清晰，首页没有总览入口
- 各功能页视觉语言不统一，卡片、列表、状态、详情面板风格分散
- 当前 UI 更像内部 CRUD 面板，不像 AI 视频生产中台

本次改版选择 `A` 方案：在不更换技术栈、不重写路由体系、不大改后端接口的前提下，把当前前端升级为深色生产控制台。

## 目标

- 保留现有 React 单页结构和现有数据流
- 新增一个默认首页 `Overview`
- 统一深色工作台框架、卡片系统和状态视觉
- 强化项目、分镜、素材、任务这四个核心页面的工作流感
- 保证实现工程化、可维护、稳定，不引入重型 UI 框架迁移风险

## 非目标

- 不迁移到 Tailwind / Radix / 多页面路由架构
- 不重写现有 API 协议和数据模型
- 不在本次改版中大幅增加新业务能力
- 不追求完全复刻参考稿，只提取适合当前仓库的视觉语言

## 信息架构

左侧导航改为 9 个入口：

1. `Overview`
2. `项目`
3. `分镜`
4. `素材`
5. `模板`
6. `生成`
7. `任务`
8. `日志`
9. `设置`

说明：

- `Overview` 作为默认首屏，用来承接总览和快捷入口
- 原有 8 个业务标签页保留，避免用户迁移成本
- 导航保持单页切换，不引入 React Router 重构

## 全局布局

整体布局调整为三段式：

- 左侧固定深色侧边栏：品牌、主导航、系统入口
- 顶部全局栏：当前项目、搜索占位、运行状态、主操作按钮
- 主内容区：统一卡片化内容容器，支持总览、列表、表单、详情面板

全局设计原则：

- 以深灰蓝为底色，亮色按钮和状态色做强调
- 所有页面统一使用圆角卡片、柔和边框和轻量高光
- 桌面体验优先，移动端保持基本可用

## 视觉系统

### 配色

- 主背景：深色控制台底色
- 次级背景：卡片层、侧栏层、面板层
- 主强调色：靛蓝/蓝紫系
- 成功/失败/运行/等待状态保留高识别色

### 状态色

任务状态：

- `pending`
- `running`
- `succeeded`
- `failed`
- `cancelled`

分镜状态：

- `draft`
- `image_selected`
- `approved`
- `rejected`

要求：

- 所有状态在 badge、卡片边框、选中态中保持一致语义
- 不同页面不再各自定义一套状态观感

### 组件风格

统一抽象以下视觉模块：

- `shell`：页面外壳、侧边栏、顶栏、内容容器
- `surface card`：统一卡片容器
- `metric card`：概览指标卡
- `section header`：标题、副标题、右侧工具区
- `status badge`：状态标签
- `action button`：主按钮、次按钮、危险按钮
- `detail panel`：详情面板、日志面板
- `result tile`：素材和结果选择网格

## 页面设计

### Overview

新增默认首页，展示：

- 当前项目摘要
- 项目/分镜/素材/任务总量指标
- 正在运行的任务队列
- 最近项目列表
- 快捷入口：新建项目、进入分镜、进入生成、查看任务

目的：

- 避免用户一进来就落在表单页面
- 让整个平台像“生产驾驶舱”

### 项目

现有“新建项目 + 项目列表”重构为：

- 左侧：项目统计与新建/编辑表单
- 右侧：项目卡片列表
- 卡片中突出项目描述、关联数量、快捷操作

### 分镜

这是最核心的工作页，重点改造：

- 提升镜头卡片的可读性
- 强化参考图、角色标签、状态标识
- 生图结果、生视频结果改成更像结果筛选器的网格
- 把“编辑、生成、通过、废弃”等操作集中到更清晰的动作条

### 素材

目标是更像素材库而不是简单卡片列表：

- 提升素材网格视觉
- 保留类型筛选
- 强化详情侧信息，包括来源、模型、提示词、参数、上游链路

### 模板

模板页保留当前功能，但统一样式：

- 列表项卡片化
- 变量填写区域更清晰
- 模板内容展示改为更稳定的代码/文本块样式

### 生成

生图和生视频保持并列双面板，但优化为控制台布局：

- 顶部统一 provider 选择与关键参数
- 文本输入区更突出
- 参数区层级更规整
- 主要提交按钮更明显

### 任务

任务页强调运行状态和结果追踪：

- 任务列表更像“队列”
- 详情页保留提示词、参数、结果、日志
- 对运行中/失败任务的操作更清楚

### 日志与设置

- `日志` 继续承接统计与调用记录
- `设置` 保留 provider 与本地路径展示
- 重点是统一信息排版与视觉系统，不扩展新功能

## 实施策略

### 第一阶段：重构骨架

- 新增 `Overview` tab
- 重构 `App` 顶层布局
- 替换全局主题和导航结构
- 建立统一卡片、指标、状态、面板样式

### 第二阶段：逐页迁移

- 项目页
- 分镜页
- 素材页
- 模板页
- 生成页
- 任务页
- 日志页
- 设置页

### 第三阶段：细节强化

- 优化空状态
- 优化选中态和 hover 态
- 统一按钮优先级
- 修正密度、换行、滚动区域与响应式细节

## 技术方案

- 继续使用现有 `React + Vite`
- 不新增 UI 依赖
- 通过拆分 `App.jsx` 内部渲染结构与抽取通用组件提升可维护性
- 样式仍以 `frontend/src/styles.css` 为主，但会重构命名和结构

建议的代码演进方向：

- `App.jsx` 保留数据与状态编排
- 逐步抽出视图级组件，至少包括：
  - `Sidebar`
  - `Topbar`
  - `Overview`
  - `ProjectView`
  - `ShotView`
  - `AssetView`
  - `TaskView`
- 保持现有 API client 不变

## 风险与控制

- 风险：单文件 `App.jsx` 改动面较大
  - 控制：先抽公共 UI 壳层，再逐页迁移
- 风险：深色主题影响可读性
  - 控制：优先保证文本对比度和状态色清晰度
- 风险：一次性大改导致局部回归
  - 控制：保持接口逻辑不动，只替换展示结构和样式

## 验收标准

- 能正常显示新增 `Overview` 首页
- 原有 8 个业务页功能仍可用
- 当前本地运行方式不变
- 项目、分镜、素材、生成、任务页面整体升级为统一深色工作台风格
- 关键操作不丢失：创建、编辑、上传、生成、选中、重试、取消、查看详情
- 页面在桌面分辨率下有明显更强的信息层级和工作台感
